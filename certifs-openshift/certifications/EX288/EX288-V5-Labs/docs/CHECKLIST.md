# Checklist de readiness

## Fondations
- [ ] Je crée rapidement projet, déploiement, service, route.
- [ ] Je sais exposer une application et vérifier la route.
- [ ] Je sais ajouter readiness / liveness / startup probes.

## Configuration
- [ ] Je sais créer et consommer ConfigMaps et Secrets.
- [ ] Je sais injecter via env, envFrom, volume, subPath.

## Builds et images
- [ ] Je sais créer ImageStream et BuildConfig.
- [ ] Je sais lancer un build binaire `--from-dir`.
- [ ] Je sais utiliser un Containerfile.
- [ ] Je sais lire l'état d'un build et corriger un échec.

## S2I
- [ ] Je sais personnaliser `assemble` et `run`.
- [ ] Je sais déclencher un build S2I et vérifier le résultat.

## Déploiement composé
- [ ] Je sais utiliser un Template OpenShift paramétré.
- [ ] Je sais déployer un chart Helm.
- [ ] Je sais appliquer une surcouche Kustomize.

## Pipelines
- [ ] Je sais appliquer un pipeline Tekton simple.
- [ ] Je sais corriger une Task ou un paramètre cassé.

## Diagnostic
- [ ] Je sais remonter rapidement de l'erreur au correctif.
- [ ] Je note toujours symptôme → commande → correction.

## Compléments utiles
- [ ] Je sais pousser vers le registry interne.
- [ ] Je sais corriger un souci RBAC simple.
- [ ] Je sais utiliser un initContainer.
