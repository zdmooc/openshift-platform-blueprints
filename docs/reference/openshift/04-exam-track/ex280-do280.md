# EX280 / DO280 - Red Hat Certified Specialist in OpenShift Administration

## Objectif

Cette section résume les compétences clés couvertes par la certification **EX280**, qui valide la capacité à installer, configurer et gérer un cluster Red Hat OpenShift Container Platform. Le cours associé est le **DO280**.

## Domaines de Compétences Clés

L\'examen se concentre sur les tâches d'administration fondamentales d\'un cluster OpenShift.

### 1. Gestion du Cluster OpenShift

- **Comprendre l'architecture d'OpenShift** : Rôle des nœuds master et worker, etcd, opérateurs.
- **Gérer la configuration du cluster** : Utiliser les opérateurs pour configurer les composants du cluster (authentification, registre, etc.).
- **Gérer les machines** : Ajouter, supprimer et gérer des nœuds worker à l'aide du Machine API.
- **Mettre à jour le cluster** : Comprendre le processus de mise à jour et gérer les canaux de mise à jour.

### 2. Gestion des Utilisateurs et des Politiques

- **Configurer l'authentification** : Configurer un fournisseur d'identité (par ex., HTPasswd, LDAP).
- **Gérer les utilisateurs et les groupes** : Créer et gérer des objets User et Group.
- **Gérer le RBAC (Role-Based Access Control)** : Créer et assigner des rôles (Roles, ClusterRoles) et des liaisons de rôles (RoleBindings, ClusterRoleBindings) pour contrôler l'accès.
- **Gérer la sécurité des pods** : Comprendre et gérer les Security Context Constraints (SCC).

### 3. Gestion des Applications

- **Déployer des applications** : Déployer des applications à partir d'images, de Dockerfiles et de templates.
- **Gérer les builds** : Créer et gérer des configurations de build (BuildConfig) pour automatiser la création d'images.
- **Gérer les déploiements** : Utiliser des DeploymentConfigs et des Deployments pour gérer le cycle de vie des applications.
- **Configurer le réseau des applications** : Créer et gérer des services et des routes pour exposer les applications.
- **Gérer le stockage des applications** : Comprendre les PersistentVolumes (PV), les PersistentVolumeClaims (PVC) et les StorageClasses pour fournir un stockage persistant.

### 4. Gestion du Stockage et du Registre

- **Configurer le stockage persistant** : Configurer une StorageClass pour le provisionnement dynamique de volumes.
- **Gérer le registre d'images interne** : Configurer et sécuriser le registre d'images intégré à OpenShift.

## À retenir / Pièges fréquents

- **La pratique est essentielle** : C'est un examen pratique. Vous devez être capable d'effectuer les tâches rapidement et efficacement en ligne de commande (`oc`).
- **Maîtriser `oc`** : La maîtrise de la CLI `oc` est absolument cruciale. Entraînez-vous à utiliser `oc create`, `oc get`, `oc describe`, `oc edit`, `oc patch`, etc.
- **Comprendre les opérateurs** : Une grande partie de la configuration du cluster se fait en modifiant les ressources personnalisées (CR) des opérateurs. Sachez comment trouver et modifier ces CR.
- **RBAC et SCC sont des points clés** : La gestion des permissions est un sujet fondamental. Assurez-vous de bien comprendre comment fonctionnent le RBAC et les SCC.
