# EX480 / DO480 - Red Hat Certified Specialist in OpenShift Data Foundation

## Objectif

Cette section résume les compétences clés couvertes par la certification **EX480**, qui valide la capacité à installer, configurer et gérer Red Hat OpenShift Data Foundation (anciennement OpenShift Container Storage) sur un cluster OpenShift. Le cours associé est le **DO480**.

## Domaines de Compétences Clés

L'examen se concentre sur le déploiement et la gestion d'une solution de stockage définie par logiciel pour les applications conteneurisées.

### 1. Déploiement d'OpenShift Data Foundation (ODF)

- **Comprendre l'architecture d'ODF** : Rôle de Ceph, Rook, NooBaa.
- **Déployer ODF** : Installer l'opérateur ODF et déployer un `StorageSystem`.
- **Configurer les pools de stockage** : Créer et gérer des pools de stockage Ceph pour différents types de charges de travail (fichiers, blocs, objets).

### 2. Configuration du Stockage

- **Provisionner du stockage bloc (RBD)** : Créer des `StorageClasses` pour le provisionnement dynamique de volumes en mode `ReadWriteOnce` (RWO).
- **Provisionner du stockage fichier (CephFS)** : Créer des `StorageClasses` pour le provisionnement dynamique de volumes en mode `ReadWriteMany` (RWX).
- **Provisionner du stockage objet (RGW/NooBaa)** : Créer des `ObjectBucketClaims` (OBC) pour fournir un stockage compatible S3 aux applications.

### 3. Gestion et Surveillance d'ODF

- **Surveiller la santé du cluster de stockage** : Utiliser les tableaux de bord ODF et Prometheus pour surveiller la capacité, les performances et l'état du cluster Ceph.
- **Étendre la capacité de stockage** : Ajouter de nouveaux disques ou de nouveaux nœuds de stockage au cluster ODF.
- **Gérer les snapshots** : Créer et restaurer des snapshots de volumes persistants.
- **Dépanner les problèmes courants** : Diagnostiquer et résoudre les problèmes liés au stockage.

### 4. Intégration avec les Applications

- **Fournir du stockage aux applications** : Créer des PVCs et des OBCs pour que les applications puissent consommer du stockage ODF.
- **Comprendre les cas d'usage** : Savoir quand utiliser le stockage bloc, fichier ou objet pour différents types d'applications.

## À retenir / Pièges fréquents

- **ODF est un produit complexe** : ODF est basé sur Ceph, qui est une solution de stockage distribué puissante mais complexe. Une bonne compréhension des concepts de base de Ceph est utile.
- **La performance dépend du matériel** : Les performances de votre stockage ODF dépendront fortement du matériel sous-jacent (disques SSD/NVMe, réseau rapide).
- **L'installation est basée sur un opérateur** : Tout le déploiement et la configuration d'ODF se font via l'opérateur ODF et ses ressources personnalisées.
- **Trois types de stockage** : Assurez-vous de bien comprendre les différences et les cas d'usage pour le stockage bloc (RWO), fichier (RWX) et objet (S3).
