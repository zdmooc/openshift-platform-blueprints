# EX288 / DO288 - Red Hat Certified Specialist in OpenShift Application Development

## Objectif

Cette section résume les compétences clés couvertes par la certification **EX288**, qui valide la capacité à développer, déployer et dépanner des applications sur un cluster Red Hat OpenShift Container Platform. Le cours associé est le **DO288**.

## Domaines de Compétences Clés

L'examen se concentre sur le cycle de vie du développement d'applications sur OpenShift.

### 1. Déploiement d'Applications

- **Déployer des applications à partir d'images de conteneurs** : Utiliser `oc new-app` et `oc create deployment`.
- **Déployer des applications à partir de code source** : Utiliser les stratégies de build Source-to-Image (S2I) pour créer des images à partir de code source (par ex., Git).
- **Déployer des applications à partir de Dockerfiles** : Utiliser la stratégie de build Docker.
- **Gérer les builds** : Dépanner les builds, utiliser les hooks de build.

### 2. Configuration des Applications

- **Gérer les ressources des applications** : Configurer les requêtes et les limites de CPU et de mémoire.
- **Utiliser les ConfigMaps et les Secrets** : Externaliser la configuration et les données sensibles des images de conteneurs.
- **Gérer la santé des applications** : Configurer les sondes de vivacité (liveness), de préparation (readiness) et de démarrage (startup).
- **Gérer le stockage persistant** : Demander du stockage persistant à l'aide de PersistentVolumeClaims (PVC).

### 3. Exposition des Applications

- **Créer des Services** : Exposer des pods au sein du cluster.
- **Créer des Routes** : Exposer des services à l'extérieur du cluster. Configurer des routes sécurisées (TLS) et des routes avec des chemins.

### 4. Dépannage des Applications

- **Analyser les logs des applications** : Utiliser `oc logs` pour inspecter les logs des pods.
- **Se connecter aux pods** : Utiliser `oc rsh` et `oc exec` pour ouvrir un shell dans un conteneur.
- **Inspecter les ressources** : Utiliser `oc get`, `oc describe` et `oc get events` pour diagnostiquer les problèmes.

### 5. Utilisation des Templates

- **Créer et utiliser des templates** : Créer des templates OpenShift pour standardiser le déploiement d'applications. Utiliser les paramètres des templates.

## À retenir / Pièges fréquents

- **C'est un examen pour les développeurs** : L'accent est mis sur l'application, pas sur l'administration du cluster. Vous n'aurez pas à configurer de fournisseurs d'identité ou à gérer des nœuds.
- **Maîtriser S2I** : La stratégie de build Source-to-Image est un concept clé d'OpenShift et est fortement testée.
- **Savoir déboguer** : Une grande partie de l'examen consiste à diagnostiquer et à résoudre des problèmes dans des applications existantes. Entraînez-vous à suivre une méthodologie de dépannage (vérifier les événements, les logs, l'état des pods, etc.).
- **Connaître les différents types de déploiement** : Comprenez la différence entre un `Deployment` et un `DeploymentConfig`.
